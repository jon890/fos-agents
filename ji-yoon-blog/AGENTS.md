# AGENTS.md — 지윤블로그 워크스페이스

이 워크스페이스는 지융로그 네이버 블로그 운영을 위한 독립 작업 영역이다.

## 주 사용자

이 워크스페이스의 주 사용자는 지융이다.
`#지윤블로그` 채널에서 지융이 요청하면 지융에게 직접 답하고 지융로그 콘텐츠 방향은 지융의 지시를 우선한다.
비포스는 워크스페이스와 자동화 설정을 함께 관리하는 사용자다.

## 범위

이 워크스페이스에서 다루는 작업은 다음이다.

- 지융로그 페르소나 분석과 유지.
- 네이버 블로그 글쓰기 스킬 자료 관리.
- 카테고리별 글쓰기 스타일 관리.
- 블로그 내부 트렌드와 외부 트렌드 분석.
- 글감 추천, 제목 후보, 본문 초안, 태그, 사진 배치 가이드 작성.
- 네이버 블로그 미리보기 또는 임시저장 자동화 준비.

## 디렉터리

- `data/naver-blog/`: 수집한 블로그 목록, 샘플 본문, 분석용 원자료.
- `references/`: 페르소나, 카테고리 스타일, 미리보기 자동화 지침.
- `drafts/`: 작성 중인 글 초안.
- `reports/`: 트렌드 분석, 내부 성과 분석, 추천 리포트.
- `scripts/`: 수집과 분석 자동화 스크립트.

## 보안 규칙

공통 보안 규칙은 상위 `../AGENTS.md`의 `작업 경계`를 따른다.
이 파일에는 지융로그 운영에만 해당하는 추가 규칙만 둔다.

네이버 계정 정보는 채팅에 쓰지 않는다.
비밀번호, 일회용 인증번호, 쿠키, 세션 값은 요청하지 않고 저장하지 않는다.
로그인이 필요하면 QR 로그인 또는 사용자의 직접 브라우저 로그인을 우선한다.

## 네이버 블로그 자동화

기본 동작은 미리보기 또는 임시저장이다.
발행은 사용자가 명시적으로 요청하고 최종 확인한 경우에만 한다.

자동화가 로그인, 보안 확인, 캡차, 기기 인증에서 막히면 중단하고 사용자에게 직접 조작을 요청한다.
우회 시도는 하지 않는다.

## 글쓰기 원칙

`naver-blog-writing` 스킬과 `references/ji-yung-persona.md`를 먼저 참고한다.
카테고리에 따라 `references/category-style-map.md`의 모듈을 적용한다.

지융로그 글은 다음을 지킨다.

- 짧은 문단.
- 사진 중심 흐름.
- 실제 경험 기반.
- 자연스러운 검색어 배치.
- AI가 수집하기 쉬운 고유명사와 실용 정보.
- 과한 광고 문구나 AI스러운 구조 금지.

## 트렌드 분석 원칙

`naver-blog-trend-research` 스킬을 사용한다.
내부 블로그 신호와 외부 트렌드를 함께 본다.
맛집, 카페, 호텔, 여행, 제품후기, 주식, 아파트 등 20-30대 관심 주제를 비교한다.

주식과 아파트 글은 투자 조언처럼 단정하지 않는다.
날짜와 출처가 있는 정보만 사용하고, 리스크를 함께 적는다.
