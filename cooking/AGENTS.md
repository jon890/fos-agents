# AGENTS.md — cooking 워크스페이스

`~/ai-nodes` 아래 독립 요리 작업 워크스페이스.
모든 에이전트는 이 파일을 먼저 읽고, 실제 산출물은 이 워크스페이스 안에 남긴다.
`CLAUDE.md`는 이 파일의 심링크다.

## 1. 목적

집에서 해볼 요리를 고를 때 필요한 자료를 모으고, 재료를 비교하고, 첫 시도용 조리 흐름을 정리한다.
레시피보다 재료 선택이 결과를 크게 좌우하는 메뉴를 우선 다룬다.

## 2. 5문서 라우팅

| 문서 | 무엇이 들어 있는지 | 언제 보는지 |
|---|---|---|
| `docs/prd.md` | 제품 범위, MVP 기능, 성공 기준 | 새 기능 추가 / 우선순위 결정 |
| `docs/data-schema.md` | config, data, logs, .env 구조 | 데이터 파일 변경 / 새 config 도입 |
| `docs/flow.md` | 조사부터 보고서 작성까지 흐름 | 흐름 추가 / 디버깅 |
| `docs/code-architecture.md` | 디렉터리 구조와 skill 경계 | 코드 구조 변경 / 새 스킬 추가 |
| `docs/adr.md` | cooking 한정 결정 기록 | 결정의 이유 확인 |

## 3. 현재 진입점

```bash
claude -p "/cooking-research <메뉴명>"
```

OpenClaw에서는 `~/.openclaw/workspace/skills/cooking-research` wrapper를 통해 이 워크스페이스로 라우팅한다.

## 4. 산출물 정책

- 메뉴별 자료는 `data/<menu-slug>/` 아래에 둔다.
- 원본 링크, 재료 후보, 비교 결과, 첫 시도 레시피를 분리한다.
- 개인 구매처나 취향이 들어간 자료는 기본적으로 private 성격으로 다룬다.
- 공개 brain에 넣고 싶을 때는 개인 맥락을 제거한 범용 지식만 별도로 정리한다.

## 5. 운영 원칙

- 현재 가격, 판매 여부, 영상 링크는 바뀔 수 있으므로 필요할 때 최신 확인을 한다.
- 재료 비교는 맛, 접근성, 가격, 실패율을 함께 본다.
- 첫 시도용 레시피는 단순하게 유지한다.
- 대체재는 가능하지만, 정통 메뉴와 얼마나 달라지는지 명시한다.
