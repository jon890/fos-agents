# career-os 출력 정책

career-os skill이 Markdown 리포트, 지원 패키지, 공개 공부 자료, 면접 자산을 만들 때 공통으로 따르는 정책이다.
각 skill의 `SKILL.md`에는 이 문서와 다른 점만 남긴다.

## 공통 원칙

- 한국어 우선 섹션 제목과 자연스러운 한국어 문장을 사용한다.
  코드 식별자, 상태값, 파일 경로, 공식 역할명은 원문을 유지한다.
- 첫 10줄 안에 결론, 목적, 우선순위, 또는 사용자가 바로 할 행동을 둔다.
- 내부 분석과 외부에 옮길 수 있는 문구를 분리한다.
  제출용 문구, 공개 공부 자료, Discord 요약에는 내부 파일 경로, plan 번호, commit hash, runner 상태를 넣지 않는다.
- 근거가 부족한 항목은 raw marker로 남기지 않는다.
  `needs_evidence` 같은 내부 판단은 `보강 필요 / 선택지 / 권장 행동` 구조로 바꾼다.
- 실제 외부 제출, 로그인, 이메일, 공개 발행, 채용 사이트 입력, candidate-profile 수정은 사용자 승인 전에는 실행하지 않는다.
  필요한 경우 `사용자 승인 필요` 항목으로만 안내한다.

## 공개 산출물

- 공개 `fos-study` 글에는 후보자 private 맥락, 특정 회사 지원 전략, reviewer 판단을 넣지 않는다.
- 사용자가 검토 전 초안 생성을 요청한 공개 글은 제목에 `[초안]`을 유지한다.
  사용자가 직접 읽고 최종화 또는 발행을 승인한 뒤에만 제거한다.
- 특정 공고, 면접, 지원서와 연결한 해석은 공개 글이 아니라 `career-os/data/` 또는 `private/` 아래 비공개 산출물에 둔다.

## 비공개 산출물

- 비공개 리포트에는 판단 근거 경로와 위험 신호를 유지한다.
- 제출용으로 옮길 수 있는 문장은 내부 근거와 분리해서 작성한다.
- Discord 요약 후보는 6줄 안팎으로 제한하고, private 지원 전략이나 이력서 문구를 그대로 노출하지 않는다.
