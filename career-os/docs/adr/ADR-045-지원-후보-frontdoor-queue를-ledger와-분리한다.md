## ADR-045 — 지원 후보 frontdoor queue를 ledger와 분리한다

- Status: Accepted
- Date: 2026-06-06

### 맥락

`/position-recommender`는 활성 공고 수집과 추천 순위를 만들 수 있고, `ingest-position-report`는 추천 결과를 ledger로 넣을 수 있다.
사용자가 원하는 경험은 "추천 후보 순위 확인 → 사용자가 준비 시작 선택 → 선택된 공고만 상세 분석·지원 준비로 승격"이다.
기존 ledger는 실제 지원 준비가 시작된 공고 원장과 agent 실행 상태가 섞여 있었다.
추천 후보를 바로 ledger에 넣으면 사용자가 고르기 전에 상세 분석 흐름으로 넘어갈 수 있고, "아직 선택 전 후보" 상태가 선명하지 않다.

### 결정

- `ledger`는 실제 지원 준비가 시작된 공고 원장으로 유지한다.
- 추천 후보 순위와 사용자 선택 대기 상태는 별도 frontdoor queue 파일에 저장한다.
- 사용자가 명시적으로 준비 시작을 선택한 후보만 ledger로 승격한다.
- 승격 후 자동 생성 범위는 상세 공고 분석, fit/gap 분석, 공부 우선순위, 예상 면접 질문까지로 제한한다.
- 최종 지원 패키지 생성, 제출 승인, 외부 사이트 입력/전송은 사용자 검토 절차를 유지한다.
- Next.js 대시보드와 관리자 로그인은 별도 plan 범위로 분리한다.

### 거절한 대안

- 추천 후보를 바로 ledger에 넣기: 사용자 선택 전 단계가 지원 준비 단계와 섞인다. "아직 선택 전" 상태를 별도로 표현할 방법이 없다.

### 결과

- 추천 후보와 실제 지원 준비 공고의 책임이 분리된다.
- 사용자는 항상 앞단에서 어떤 공고를 시작할지 선택할 수 있다.
- application-flow-agent는 사용자가 승인한 후보만 상세 분석·지원 준비 흐름으로 처리한다.
- plan039 대시보드는 frontdoor queue와 ledger를 함께 읽는 읽기 전용 MVP 위에 얹을 수 있다.
