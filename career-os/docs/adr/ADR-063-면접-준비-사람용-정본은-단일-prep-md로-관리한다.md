## ADR-063 — 면접 준비 사람용 정본은 단일 prep.md로 관리한다

- Status: Accepted
- Date: 2026-06-08

### 맥락

포지션별 면접 준비 내용이 예상 질문 드릴, 면접 준비 리포트, 전략, 체크리스트, 학습 재료 등으로 흩어져 있었다.
dashboard와 사람이 읽는 흐름 모두 복잡해진다.
사용자는 지금 읽고 답변 연습할 하나의 정제된 문서를 원하며, 기계가 누적해야 하는 데이터는 분리해도 되지만 사람이 보는 준비 자산은 하나로 합치는 편이 낫다.

### 결정

- 포지션별 면접 준비의 사람용 정본은 `private/<company-slug>/<position-slug>/interview/prep.md` 하나로 둔다.
- `prep.md`는 오늘의 면접 준비 요약, 예상 질문 드릴, 전략, 체크리스트, 단기 준비 항목, 다음 액션을 섹션으로 포함한다.
- 기존 개별 파일들은 dashboard의 primary asset이 아니므로 `prep.md`로 정제·흡수하고 legacy reference로 정리한다.
- 답변 기록과 피드백 로그는 누적 데이터이므로 계속 분리한다.
- 날짜별 snapshot은 기본 생성하지 않으며, 추적이 필요할 때만 선택적으로 만든다.
- dashboard는 면접 hub 상단에서 `prep.md`를 우선 보여주고, 예상 질문도 `prep.md` 섹션에서 파싱한다.
- 공개 공부팩은 `prep.md`의 개인 답변·지원 전략·민감 맥락을 그대로 복사하지 않고 public-safe로 재작성한다.

### 결과

- 면접 준비 자료가 사람이 읽는 한 문서로 정리된다.
- generator와 processor는 여러 산출물을 흩뿌리는 대신 `prep.md` 갱신을 중심으로 동작한다.
- 답변/피드백 로그는 유지되어 연습 이력과 사람용 준비 문서의 책임이 분리된다.
